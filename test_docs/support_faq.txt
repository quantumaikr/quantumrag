QuantumRAG 자주 묻는 질문 (FAQ)

=== 설치 및 설정 ===

Q1: QuantumRAG 설치 최소 요구 사양은?
A1: Python 3.11 이상, RAM 8GB 이상, 디스크 20GB 이상이 필요합니다. GPU는 선택사항이나 대규모 인제스트 시 권장됩니다.

Q2: API 키 없이 사용할 수 있나요?
A2: 네, Ollama를 사용하면 로컬에서 API 키 없이 실행 가능합니다. 다만 OpenAI나 Gemini 등 클라우드 모델 사용 시에는 해당 서비스의 API 키가 필요합니다.

Q3: Docker로 배포할 때 환경 변수는 어떻게 설정하나요?
A3: docker run 시 -e 옵션으로 설정합니다. 예: docker run -e OPENAI_API_KEY=sk-xxx -e QUANTUMRAG_LANGUAGE=ko quantumrag:latest

=== 문서 인제스트 ===

Q4: 한 번에 인제스트할 수 있는 최대 문서 수는?
A4: 기술적 제한은 없으나, 메모리 사용량에 따라 달라집니다. 일반적으로 8GB RAM 환경에서 약 5,000개 문서(총 1GB 이하)를 안정적으로 처리할 수 있습니다. 대규모 처리 시 배치 인제스트를 권장합니다.

Q5: 인제스트 중 오류가 발생하면 어떻게 되나요?
A5: 개별 문서의 파싱 오류는 해당 문서만 건너뛰고 나머지를 계속 처리합니다. 오류 내역은 IngestResult.errors에 기록됩니다.

Q6: 이미 인제스트한 문서를 업데이트하려면?
A6: 현재는 같은 경로의 문서를 다시 인제스트하면 중복 저장됩니다. 향후 v3.0에서 자동 중복 감지 및 증분 업데이트를 지원할 예정입니다.

Q7: 스캔된 PDF(이미지 PDF)도 처리 가능한가요?
A7: 현재는 텍스트 추출이 가능한 PDF만 지원합니다. 스캔된 PDF의 경우 별도 OCR 처리 후 텍스트 파일로 변환하여 인제스트해야 합니다. v3.0에서 OCR 통합을 계획하고 있습니다.

=== 검색 및 질의 ===

Q8: 응답 속도가 느린 경우 어떻게 개선할 수 있나요?
A8: 1) SIMPLE 쿼리 라우팅 확인 (단순 질문은 자동으로 경량 모델 사용), 2) top_k 값 줄이기 (기본 5 → 3), 3) 리랭킹 비활성화 (rerank=False), 4) gpt-5.4-nano 등 경량 모델 사용.

Q9: 한국어와 영어가 섞인 문서를 잘 처리하나요?
A9: 네, 자동 언어 감지와 Kiwi 한국어 형태소 분석기를 통해 한영 혼합 문서를 효과적으로 처리합니다. 다만 BM25 검색에서 영어 키워드 매칭이 약할 수 있어, 벡터 검색이 보완합니다.

Q10: confidence가 insufficient_evidence로 나오는 이유는?
A10: 주로 세 가지 경우입니다: 1) 관련 문서가 인제스트되지 않은 경우, 2) 질문이 너무 모호한 경우, 3) 검색된 청크와 질문의 관련도가 임계값 미만인 경우. 관련 문서를 추가하거나 질문을 구체화하면 개선됩니다.

=== 비용 및 성능 ===

Q11: 월 비용이 얼마나 드나요?
A11: 문서 1,000개, 일 500회 쿼리 기준 약 15~25만원(OpenAI API 비용)입니다. Ollama 로컬 모델 사용 시 API 비용은 0원이나 서버 비용이 발생합니다.

Q12: 임베딩 비용을 줄이는 방법은?
A12: 1) text-embedding-3-small (1536차원) 사용 시 text-embedding-3-large 대비 약 5배 저렴, 2) 청크 크기를 512에서 1024로 늘리면 청크 수가 줄어 비용 절감, 3) 캐싱 활성화.

=== 보안 ===

Q13: 데이터가 외부로 전송되나요?
A13: 클라우드 모델(OpenAI, Gemini 등) 사용 시 쿼리와 컨텍스트가 해당 API 서버로 전송됩니다. 데이터 유출이 우려되는 경우 Ollama를 사용한 로컬 실행을 권장합니다.

Q14: 온프레미스 배포가 가능한가요?
A14: 현재는 공식적으로 지원하지 않으나 2025년 Q2에 온프레미스 버전 출시를 계획하고 있습니다. 조기 접근을 원하시면 enterprise@quantumsoft.ai로 문의하세요.

=== 기타 ===

Q15: Slack이나 Teams와 연동하려면?
A15: QuantumChat 제품이 Slack, Teams 연동을 지원합니다. QuantumRAG 단독으로는 REST API를 통한 커스텀 연동이 필요합니다.

Q16: 경쟁사 대비 장점은?
A16: 1) Triple Index Fusion으로 업계 최고 검색 정확도, 2) 한국어 특화 처리 (HWP, 형태소 분석), 3) 적응형 쿼리 라우팅으로 비용 효율성, 4) 높은 고객 만족도 (NPS 72).
