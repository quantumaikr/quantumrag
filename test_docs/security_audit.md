# QuantumRAG 보안 감사 보고서

> 감사 기간: 2024-08-01 ~ 2024-08-31
> 감사 수행: 외부 보안 컨설팅 업체 (사이버가드)
> 대상 시스템: QuantumRAG v2.4.2, QuantumChat v1.3

## 1. 감사 범위

- API 서버 보안 (인증/인가)
- 데이터 암호화 (전송 중/저장 시)
- 인프라 보안 (AWS 환경)
- 코드 보안 (SAST/DAST)
- 접근 제어 및 감사 로그

## 2. 발견 사항 요약

| 등급 | 건수 | 조치 기한 |
|------|------|-----------|
| 심각 (Critical) | 1건 | 즉시 (7일 이내) |
| 높음 (High) | 3건 | 30일 이내 |
| 중간 (Medium) | 5건 | 90일 이내 |
| 낮음 (Low) | 4건 | 다음 릴리스 |
| **총계** | **13건** | - |

## 3. 심각 사항

### [SEC-001] API 키 평문 저장 (Critical)

**설명**: QuantumRAG 설정 파일(quantumrag.yaml)에 OpenAI API 키가 평문으로 저장됨. 서버 침해 시 API 키 유출 가능.

**영향**: API 키 탈취 시 무단 API 사용, 비용 발생, 데이터 유출 가능

**권고 조치**:
1. 환경 변수 또는 시크릿 매니저(AWS Secrets Manager) 사용
2. 설정 파일에서 API 키 직접 기재 금지
3. API 키 로테이션 정책 수립 (90일 주기)

**조치 상태**: 2024-09-15 완료 — 환경 변수 방식으로 전환, 기존 키 폐기 및 재발급

## 4. 높음 사항

### [SEC-002] 인증 토큰 만료 시간 과다 (High)

**설명**: JWT 토큰 만료 시간이 720시간(30일)으로 설정됨. 토큰 탈취 시 장기간 무단 접근 가능.

**권고**: 액세스 토큰 15분, 리프레시 토큰 7일로 조정

**조치 상태**: 2024-10-01 완료

### [SEC-003] SQL Injection 취약점 (High)

**설명**: 메타데이터 필터링 API에서 사용자 입력값을 직접 SQL 쿼리에 삽입하는 부분 발견. `GET /api/v1/search?filter=title:' OR 1=1--` 패턴으로 공격 가능.

**영향**: 데이터베이스 전체 데이터 유출 가능

**권고**: 파라미터 바인딩 사용, ORM 쿼리 빌더 활용

**조치 상태**: 2024-09-20 완료 — 전체 API 엔드포인트 파라미터 바인딩 적용

### [SEC-004] CORS 설정 과다 허용 (High)

**설명**: API 서버의 CORS 설정이 `Access-Control-Allow-Origin: *`로 되어 있어 모든 도메인에서 API 호출 가능.

**권고**: 허용 도메인 화이트리스트 방식으로 변경

**조치 상태**: 2024-09-25 완료

## 5. 중간 사항

### [SEC-005] 로그에 민감 정보 포함 (Medium)

**설명**: 디버그 로그에 사용자 쿼리 원문과 검색 결과 일부가 포함됨. 로그 수집 시스템을 통해 민감 데이터 유출 가능.

**권고**: 프로덕션 로그 레벨 INFO 이상으로 제한, 쿼리 내용 해싱 처리

**조치 상태**: 미조치 — 2024-11-30까지 완료 예정

### [SEC-006] 파일 업로드 크기 제한 미설정 (Medium)

**설명**: 인제스트 API에 파일 크기 제한이 없어 대용량 파일로 DoS 공격 가능.

**권고**: 파일당 100MB, 요청당 500MB 제한 설정

**조치 상태**: 미조치 — 2024-11-30까지 완료 예정

### [SEC-007] TLS 1.0/1.1 지원 (Medium)

**설명**: 로드밸런서에서 TLS 1.0, 1.1 프로토콜 허용 중. 알려진 취약점 존재.

**권고**: TLS 1.2 이상만 허용

**조치 상태**: 2024-09-10 완료

### [SEC-008] Rate Limiting 미흡 (Medium)

**설명**: Free 플랜은 분당 10회, Pro 플랜은 분당 60회로 제한되어 있으나, 인증 실패 시도에 대한 별도 제한이 없음. 브루트포스 공격 가능.

**권고**: 인증 실패 5회 초과 시 15분 계정 잠금, IP 기반 제한 추가

**조치 상태**: 미조치

### [SEC-009] 의존성 취약점 (Medium)

**설명**: 사용 중인 Python 패키지 중 3개에서 알려진 취약점 발견
- `requests` 2.28.1 → CVE-2023-32681 (쿠키 유출)
- `urllib3` 1.26.12 → CVE-2023-43804 (헤더 유출)
- `cryptography` 39.0.0 → CVE-2023-49083 (DoS)

**권고**: 최신 버전으로 업데이트

**조치 상태**: 2024-09-05 완료

## 6. 낮음 사항

### [SEC-010] 에러 메시지 정보 노출 (Low)
- 500 에러 시 스택 트레이스가 응답에 포함됨
- **조치**: 프로덕션에서 디버그 모드 비활성화

### [SEC-011] 헤더 정보 노출 (Low)
- `Server: uvicorn` 헤더가 노출되어 서버 소프트웨어 식별 가능
- **조치**: 서버 헤더 제거

### [SEC-012] 비밀번호 정책 미흡 (Low)
- 관리자 콘솔 비밀번호에 최소 길이 제한만 있고, 복잡도 요구 없음
- **조치**: 대문자/소문자/숫자/특수문자 조합 요구

### [SEC-013] 세션 고정 공격 취약 (Low)
- 로그인 후 세션 ID가 변경되지 않음
- **조치**: 로그인 성공 시 새 세션 ID 발급
