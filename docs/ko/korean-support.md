# 한국어 지원

> QuantumRAG는 전체 파이프라인에서 한국어를 1등 시민으로 취급합니다.

---

## 개요

한국어는 RAG 시스템에 고유한 도전 과제를 제시합니다: 교착어 형태론, 한영 혼합 텍스트, 레거시 인코딩, 독자 문서 형식 등. QuantumRAG는 이 모든 문제를 네이티브하게 해결합니다.

| 도전 과제 | 해결책 |
|----------|--------|
| 교착어 형태론 | Kiwi 형태소 분석으로 BM25 토큰화 |
| 한영 혼합 텍스트 | 스크립트별 개별 토큰화 |
| HWP 문서 형식 | 네이티브 파서 (olefile 기반) |
| EUC-KR 레거시 인코딩 | 자동 감지 및 변환 |
| 한국어 쿼리 패턴 | 형태론 인식 라우팅 및 분해 |
| 이중 언어 사용 | 시스템 프롬프트 자동 전환 |

## 설정

### 한국어 의존성 설치

```bash
pip install quantumrag[korean]
# 또는
pip install kiwipiepy
```

### 설정

```yaml
# quantumrag.yaml
language: "ko"

korean:
  morphology: "kiwi"      # kiwi (권장) 또는 mecab
  hwp_parser: "auto"       # auto, pyhwp, libreoffice
  mixed_script: true        # 한영 혼합 텍스트 처리
```

## Kiwi 형태소 분석

[Kiwi](https://github.com/bab2min/kiwipiepy) (Korean Intelligent Word Identifier)는 한국어 텍스트에 대해 정확한 토큰화를 제공합니다.

### BM25에서 Kiwi가 중요한 이유

표준 공백 토큰화는 한국어에서 실패합니다:

```
입력:    "보안이슈의심각도가높습니다"
공백:    ["보안이슈의심각도가높습니다"]  → 1개 토큰 (검색 불가)
Kiwi:   ["보안", "이슈", "심각도", "높"]  → 4개 의미 있는 토큰
```

QuantumRAG에서 Kiwi 사용 위치:
- **BM25 인덱싱**: 인제스트 시점에 Kiwi로 청크 토큰화
- **BM25 쿼리**: 동일한 방식으로 쿼리 토큰화하여 매칭
- **Derived Index**: 한국어 동의어/계층 용어 정확 생성

### 폴백

Kiwi가 설치되지 않은 경우 정규식 기반 토큰화로 폴백:

```
패턴: [\uac00-\ud7a3]+ | [a-zA-Z]+ | [0-9]+
```

기능적이나 복합어와 활용형에서 정확도가 낮습니다.

## HWP/HWPX 문서 파싱

HWP는 한국 정부 및 관공서 문서의 표준 형식입니다.

### 지원 형식

| 형식 | 확장자 | 파서 |
|------|--------|------|
| HWP (바이너리) | `.hwp` | olefile 기반 파서 |
| HWPX (XML) | `.hwpx` | XML 기반 파서 |

### 사용법

```python
engine = Engine()
engine.ingest("./government_docs")  # .hwp 파일 자동 감지
```

별도 설정 불필요 — HWP 파서는 파일 확장자로 자동 선택됩니다.

## 한국어 쿼리 처리

### 형태론 인식 쿼리 라우팅

QuantumRAG는 한국어 고유의 쿼리 패턴을 감지하여 최적 라우팅합니다:

| 패턴 | 감지 | 동작 |
|------|------|------|
| 등급 범위 | "High 등급 이상" | 엔티티 인덱스 계층 조회 |
| 상태 필터 | "조치 완료된" | 엔티티 인덱스 속성 필터 |
| 열거 | "모두 나열해주세요" | 넓은 범위 검색 (top_k=12) |
| 비교 | "차이점은 무엇인가요" | 복잡 라우팅 |
| 시간 | "상반기 동안" | 시간 인식 검색 |

### 엔티티 감지 (한국어)

엔티티 디텍터는 한국어 패턴을 인식합니다:

```
"조치 완료된 High 등급 이상 보안 이슈의 ID를 모두 나열"
  → severity_gte: "High"
  → status: "완료"
  → domain: "security"
```

지원 한국어 패턴:
- 등급: 심각, 높음, 중간, 낮음 → Critical, High, Medium, Low
- 상태: 완료, 미조치, 미해결, 진행중, 보류
- 도메인 지시자: 보안, 특허, 버전, 고객

### 쿼리 분해

복합 한국어 질문을 독립 서브 쿼리로 분리:

```
"Series C 투자금의 가장 큰 사용 용도는 무엇이고 얼마인가요?"
  → 서브 쿼리 1: "Series C 투자금의 가장 큰 사용 용도는 무엇인가요?"
  → 서브 쿼리 2: (건너뜀 — 10자 미만으로 너무 짧음)
```

최소 10자 임계값으로 "얼마인가요?" 같은 무의미한 조각을 방지합니다.

## 이중 언어 시스템 프롬프트

QuantumRAG는 쿼리 언어에 따라 한국어 또는 영어 시스템 프롬프트를 자동 선택합니다.

### 한국어 생성 규칙

한국어 시스템 프롬프트는 11가지 전문 규칙을 포함합니다:

1. 제공된 컨텍스트에서만 답변
2. 인라인 인용 `[1]`, `[2]` 사용
3. 모든 상태 정보 포함 (확정/진행중/계획)
4. 수량/금액 질문에 모든 항목 나열
5. 수치가 다를 때 각각의 수치와 출처 제시
6. 불확실성 설명 및 상충 출처 인용
7. 컨텍스트의 구체적 숫자 포함
8. 조건부 질문에 논리적 추론 적용
9. 컨텍스트 부족 시 `INSUFFICIENT_EVIDENCE` 사용
10. 정보 조작 금지
11. 경쟁사 정보를 자사 정보와 혼동하지 않기

## 한국어용 Derived Index

인제스트 시점에 한국어 특화 검색 용어 생성:

| 원본 | 생성 용어 |
|------|----------|
| severity:Critical | "Critical 이상", "High 이상", "Medium 이상" |
| status:완료 | "조치 완료된", "해결됨" |
| 일자 2024-07-20 | "하반기", "Q3", "2024년 Q3" |
| 금액 3.2억 | "약 3억", "억 단위" |
| 버전 v2.4.0 | "v2.4", "2.4 버전" |

이 용어들은 BM25에 인덱싱되어, 정확한 표현이 달라도 한국어 쿼리가 매칭됩니다.

## 혼합 스크립트 처리

한국어 문서는 한글, 라틴, 숫자를 빈번하게 혼합합니다:

```
"QuantumSoft의 2024년 Q3 매출은 150억원입니다"
```

QuantumRAG의 처리 방식:
1. 스크립트 경계 감지
2. 세그먼트별 최적 토크나이저 적용 (한글 → Kiwi, 라틴 → 표준)
3. 크로스 스크립트 엔티티 보존 (예: "Series C", "v2.4.0")

## EUC-KR 인코딩 지원

레거시 한국어 문서는 종종 EUC-KR 인코딩을 사용합니다. QuantumRAG는:

1. 파일 내용에서 인코딩 자동 감지
2. UTF-8로 투명하게 변환
3. 잘못된 시퀀스 우아하게 처리

설정 불필요 — 파싱 중 인코딩 감지가 자동으로 수행됩니다.
