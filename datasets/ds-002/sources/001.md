# Python `typing` Module Documentation

## Overview
The `typing` module provides runtime support for type hints in Python. Type hints are not enforced at runtime but are used by static type checkers, IDEs, and linters.

## Type Aliases

### Modern Syntax (Python 3.12+)
```python
type Vector = list[float]

def scale(scalar: float, vector: Vector) -> Vector:
    return [scalar * num for num in vector]
```

### Backward Compatible Methods
```python
Vector = list[float]

from typing import TypeAlias
Vector: TypeAlias = list[float]
```

## NewType - Creating Distinct Types

```python
from typing import NewType

UserId = NewType('UserId', int)
some_id = UserId(524313)

def get_user_name(user_id: UserId) -> str: ...
```

Key Points:
- At runtime, `NewType` returns its argument unchanged: `UserId(23413) + UserId(54341)` returns `int`, not `UserId`
- Cannot subclass a `NewType`: `class AdminUserId(UserId): pass` fails
- Can create `NewType` based on another: `ProUserId = NewType('ProUserId', UserId)`

## Annotating Callable Objects

### Basic Callable Syntax
```python
from collections.abc import Callable

def feeder(get_next_item: Callable[[], str]) -> None: ...
def async_query(
    on_success: Callable[[int], None],
    on_error: Callable[[int, Exception], None]
) -> None: ...
```

### ParamSpec and Concatenate
```python
from typing import Concatenate, ParamSpec
P = ParamSpec('P')

def with_lock(f: Callable[Concatenate[Lock, P], int]) -> Callable[P, int]:
    def inner(*args: P.args, **kwargs: P.kwargs) -> int:
        return f(my_lock, *args, **kwargs)
    return inner
```

## Generics

### Type Parameter Syntax (Python 3.12+)
```python
def first[T](l: Sequence[T]) -> T:
    return l[0]
```

### Traditional TypeVar Approach
```python
T = TypeVar('T')
def second(l: Sequence[T]) -> T:
    return l[1]
```

### Generic Classes
```python
class LoggedVar[T]:
    def __init__(self, value: T, name: str, logger: Logger) -> None:
        self.name = name
        self.logger = logger
        self.value = value

    def set(self, new: T) -> None:
        self.log('Set ' + repr(self.value))
        self.value = new

    def get(self) -> T:
        self.log('Get ' + repr(self.value))
        return self.value
```

## TypeVar - Type Variables

### Bounded TypeVar
```python
S = TypeVar('S', bound=str)  # Can be any subtype of str
```

### Constrained TypeVar
```python
A = TypeVar('A', str, bytes)  # Must be exactly str or bytes

def concatenate(x: A, y: A) -> A:
    return x + y

concatenate("one", "two")    # OK, returns str
concatenate(b"one", b"two")  # OK, returns bytes
concatenate("one", b"two")   # Error: cannot mix types
```

### Variance
```python
T_co = TypeVar('T_co', covariant=True)       # Covariant (for output/return)
T_contra = TypeVar('T_contra', contravariant=True)  # Contravariant (for input)
T_inv = TypeVar('T_inv')                     # Invariant (default)
```

## Special Types

### Any
Every type is compatible with `Any`. `Any` is compatible with every type.

### Literal
```python
from typing import Literal
type Mode = Literal['r', 'rb', 'w', 'wb']

def open_helper(file: str, mode: Mode) -> str: ...
open_helper('/some/path', 'r')      # OK
open_helper('/other/path', 'typo')  # Error
```

### Never and NoReturn
```python
from typing import Never
def stop() -> Never:
    raise RuntimeError('no way')
```

### Self
```python
from typing import Self
class Foo:
    def return_self(self) -> Self:
        return self
```

## Special Forms

### Union and Optional
```python
def foo(arg: int | str) -> None: ...
def bar(arg: int | None = None) -> None: ...
```

### ClassVar
```python
from typing import ClassVar
class Starship:
    stats: ClassVar[dict[str, int]] = {}  # class variable
    damage: int = 10                       # instance variable
```

### Final
```python
from typing import Final
MAX_SIZE: Final = 9000
MAX_SIZE += 1  # Error
```

### Annotated
```python
from typing import Annotated
T1 = Annotated[int, ValueRange(-10, 5)]
```

### TypeGuard vs TypeIs

TypeGuard (older approach):
```python
def is_str_list(val: list[object]) -> TypeGuard[list[str]]:
    return all(isinstance(x, str) for x in val)
```

TypeIs (newer, more intuitive):
```python
def is_parent(val: object) -> TypeIs[Parent]:
    return isinstance(val, Parent)
```

Key Difference:
- `TypeIs` requires narrowed type to be subtype of input type
- `TypeGuard` does not have this restriction
- `TypeIs` provides intersection types on True, `TypeGuard` replaces type
- `TypeIs` narrows on False, `TypeGuard` does not

## Generic Protocols

### Basic Protocol
```python
from typing import Protocol

class Closable(Protocol):
    def close(self) -> None: ...

class File:
    def close(self) -> None: ...

def use_closable(c: Closable) -> None:
    c.close()

use_closable(File())  # OK - structural subtyping
```

### Runtime Checkable Protocol
```python
@runtime_checkable
class Closable(Protocol):
    def close(self) -> None: ...

assert isinstance(open('/some/file'), Closable)
```

## TypedDict

```python
from typing import TypedDict, Required, NotRequired

class Movie(TypedDict, total=False):
    title: Required[str]
    year: NotRequired[int]
```

## NamedTuple

```python
from typing import NamedTuple

class Employee(NamedTuple):
    name: str
    id: int = 3
```

## Utility Functions

### get_type_hints
```python
hints = get_type_hints(func)  # strips metadata
hints = get_type_hints(func, include_extras=True)  # keeps Annotated metadata
```

### get_origin and get_args
```python
get_origin(list[int])  # list
get_args(list[int])    # (int,)
get_origin(int | str)  # types.UnionType
get_args(int | str)    # (int, str)
```

### cast
```python
y: str = cast(str, func(2))  # Type checker treats y as str, no runtime effect
```

### @overload
```python
@overload
def process(x: int) -> str: ...
@overload
def process(x: str) -> int: ...
def process(x: int | str) -> int | str:
    if isinstance(x, int): return str(x)
    return len(x)
```

## Deprecated Aliases
Old `typing.Dict`, `typing.List`, `typing.Set` are deprecated in favor of `dict`, `list`, `set` built-in generics (since Python 3.9).
