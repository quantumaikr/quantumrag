# Python asyncio - Coroutines and Tasks

## Overview
High-level asyncio APIs for coroutines and Tasks.

## Coroutines
Declared using `async def` syntax. Three ways to run:
1. `asyncio.run()` — top-level entry point
2. `await` — sequential
3. `asyncio.create_task()` / `asyncio.TaskGroup` — concurrent

## TaskGroup (Python 3.11+)

```python
async with asyncio.TaskGroup() as tg:
    task1 = tg.create_task(some_coro(...))
    task2 = tg.create_task(another_coro(...))
```

Features: waits for all tasks, cancels remaining on failure, groups exceptions into `ExceptionGroup`.

## asyncio.gather(*aws, return_exceptions=False)

Runs awaitables concurrently, returns aggregated results as list.

## asyncio.shield(aw)

Protects an awaitable from cancellation.

## Timeouts

- `asyncio.timeout(delay)` — context manager with deadline
- `asyncio.wait_for(aw, timeout)` — wait with timeout

## asyncio.wait(*aws, return_when=ALL_COMPLETED)

Returns `(done, pending)` sets. Constants: `FIRST_COMPLETED`, `FIRST_EXCEPTION`, `ALL_COMPLETED`.

## asyncio.to_thread(func)

Runs blocking IO in separate thread asynchronously.

## Task Object

Key methods: `done()`, `result()`, `cancel()`, `cancelled()`, `get_name()`, `set_name()`, `get_coro()`.
