# Python asyncio - Coroutines and Tasks Documentation

## Overview

This documentation covers high-level asyncio APIs for working with coroutines and Tasks in Python's asynchronous I/O module.

## Coroutines

**Coroutines** are declared using `async def` syntax and are the preferred way to write asyncio applications.

### Basic Example

```python
import asyncio

async def main():
    print('hello')
    await asyncio.sleep(1)
    print('world')

asyncio.run(main())
```

### Three Ways to Run Coroutines

1. **Using `asyncio.run()`** - runs the top-level entry point
2. **Awaiting coroutines** - sequentially waits for each coroutine
3. **Using `asyncio.create_task()`** or **`asyncio.TaskGroup`** - runs concurrently

## Awaitables

An **awaitable** object can be used in an `await` expression. There are three main types:

### 1. Coroutines
### 2. Tasks - Used to schedule coroutines concurrently
### 3. Futures - Low-level awaitable objects representing eventual results of asynchronous operations.

## Creating Tasks

### `asyncio.create_task()`

```python
asyncio.create_task(coro, *, name=None, context=None, eager_start=None, **kwargs)
```

Wraps a coroutine into a `Task` and schedules its execution.

**Parameters:**
- `coro`: The coroutine to wrap
- `name`: Optional task name
- `context`: Custom `contextvars.Context`
- `eager_start`: If `True`, executes eagerly until first blocking point

**Important:** Save a reference to avoid garbage collection:
```python
background_tasks = set()

for i in range(10):
    task = asyncio.create_task(some_coro(param=i))
    background_tasks.add(task)
    task.add_done_callback(background_tasks.discard)
```

## Task Cancellation

Tasks can be cancelled safely. When cancelled, `asyncio.CancelledError` is raised.

```python
async def cancel_me():
    print('before sleep')
    try:
        await asyncio.sleep(3600)
    except asyncio.CancelledError:
        print('cancel sleep')
        raise
    finally:
        print('after sleep')
```

## Task Groups

### `asyncio.TaskGroup` (Python 3.11+)

An asynchronous context manager for managing a group of tasks with strong safety guarantees.

```python
async def main():
    async with asyncio.TaskGroup() as tg:
        task1 = tg.create_task(some_coro(...))
        task2 = tg.create_task(another_coro(...))
    print(f"Both tasks completed: {task1.result()}, {task2.result()}")
```

**Features:**
- Waits for all tasks when context exits
- If any task fails, cancels remaining tasks
- Groups exceptions into `ExceptionGroup`
- Preserves cancellation semantics

## Sleeping

### `asyncio.sleep(delay, result=None)`

Blocks for `delay` seconds, allowing other tasks to run.

## Running Tasks Concurrently

### `asyncio.gather(*aws, return_exceptions=False)`

Runs awaitable objects concurrently and returns aggregated results.

```python
async def factorial(name, number):
    f = 1
    for i in range(2, number + 1):
        await asyncio.sleep(1)
        f *= i
    return f

async def main():
    L = await asyncio.gather(
        factorial("A", 2),
        factorial("B", 3),
        factorial("C", 4),
    )
    print(L)  # [2, 6, 24]
```

## Eager Task Factory

### `asyncio.eager_task_factory(loop, coro, *, name=None, context=None)`

A task factory for eager task execution. Coroutines begin execution synchronously during Task construction and only schedule on the event loop if they block.

## Shielding from Cancellation

### `asyncio.shield(aw)`

Protects an awaitable from cancellation.

## Timeouts

### `asyncio.timeout(delay)`

Returns an asynchronous context manager to limit waiting time.

```python
async def main():
    try:
        async with asyncio.timeout(10):
            await long_running_task()
    except TimeoutError:
        print("The long operation timed out, but we've handled it.")
```

### `asyncio.Timeout` Class

**Methods:**
- `when()` -> float | None: Return current deadline
- `reschedule(when)`: Reschedule the timeout
- `expired()` -> bool: Check if deadline exceeded

### `asyncio.timeout_at(when)`

Similar to `timeout()` but takes absolute time instead of delay.

### `asyncio.wait_for(aw, timeout)`

Wait for awaitable with a timeout, raising `TimeoutError` if exceeded.

## Waiting Primitives

### `asyncio.wait(*aws, *, timeout=None, return_when=ALL_COMPLETED)`

Run Futures and Tasks concurrently and block until condition is met.

**Returns:** `(done, pending)` sets

**`return_when` constants:**
- `FIRST_COMPLETED`: Return when any finishes
- `FIRST_EXCEPTION`: Return when any raises exception
- `ALL_COMPLETED`: Return when all finish (default)

### `asyncio.as_completed(aws, *, timeout=None)`

Run awaitable objects and iterate over results as they finish.

## Running in Threads

### `asyncio.to_thread(func, /, *args, **kwargs)`

Asynchronously run an IO-bound function in a separate thread.

```python
def blocking_io():
    time.sleep(1)

async def main():
    await asyncio.gather(
        asyncio.to_thread(blocking_io),
        asyncio.sleep(1))
```

## Scheduling from Other Threads

### `asyncio.run_coroutine_threadsafe(coro, loop)`

Submit a coroutine to an event loop from another thread. Thread-safe.
Returns `concurrent.futures.Future`.

## Introspection

- `asyncio.current_task()`: Return the currently running Task instance
- `asyncio.all_tasks()`: Return set of not yet finished Task objects
- `asyncio.iscoroutine(obj)`: Return True if obj is a coroutine object

## Task Object

### `asyncio.Task` Class

A Future-like object that runs a Python coroutine. Not thread-safe.

### Key Methods
- `done()`: Return True if Task is done
- `result()`: Return the result
- `exception()`: Return the exception
- `cancel(msg=None)`: Request cancellation
- `cancelled()`: Return True if cancelled
- `uncancel()`: Decrement cancellation request count
- `cancelling()`: Return number of pending cancellation requests
- `add_done_callback(callback)`: Add completion callback
- `get_name()` / `set_name(value)`: Task name management
- `get_coro()`: Return the wrapped coroutine
- `get_stack()`: Return stack frames
