# Kubernetes 기반 오픈소스 MLOps 플랫폼 구축

## 개요

삼성SDS는 기업의 AI 핵심 업무 적용이 확대됨에 따라 Kubeflow 기반의 오픈소스 MLOps 플랫폼을 구축하여 활용하고 있습니다.

## 주요 통계

- 전 세계 기업의 50%가 핵심 업무에 AI를 활용 (맥킨지, 2020)
- 66%는 AI 투자를 통해 수익 증가를 보고
- 22%는 AI 배포를 통해 5% 이상의 EBIT 기여 달성
- 개발된 AI 모델의 80%는 프로덕션 전에 폐기됨
- Gartner는 2025년까지 MLOps 시장 규모를 40억 달러로 전망
- 2025년까지 기업의 50%가 AI 플랫폼을 도입할 것으로 예상
- Spotify는 Kubeflow Pipeline 도입으로 700% 생산성 향상 달성

## 핵심 과제

AI 모델이 프로덕션에 실패하는 가장 큰 장벽은 "AI 모델이 데이터 변동에 민감하게 반응"하기 때문입니다. 통제된 연구 환경의 데이터셋과 실제 프로덕션 환경의 데이터셋이 크게 다르므로, 지속적인 모니터링과 재학습이 필요합니다.

## MLOps 솔루션 구성요소

**Kubeflow** 주요 컴포넌트:
- Notebook Server (Jupyter/VSCode/RStudio)
- Katib (하이퍼파라미터 최적화)
- Training Operator (분산 학습)
- KFServing (모델 배포)
- Pipeline (엔드-투-엔드 워크플로우 자동화)

## 삼성SDS 구현 사례

### 모델 개발 단계
- 개별 리소스 할당을 통한 개발 환경 제공
- 개발 환경 프로비저닝 시간을 1주일 이상에서 2~3분으로 단축

### 모델 최적화 단계
- 공유 GPU 클러스터(Nvidia A100) 활용
- 분산 학습 작업을 위한 통합 모니터링 UI 제공

## Volcano Scheduler 혁신

분산 학습 리소스 최적 할당을 위해 다음 기술을 적용:
- **Gang Scheduling**: 분산 학습에 필요한 모든 리소스가 동시에 확보될 때까지 대기
- **Priority-based Execution**: 우선순위 기반 작업 실행
- **Bin-packing**: GPU 노드 간 최적 리소스 배치
