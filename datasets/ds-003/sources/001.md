# 트랜스포머(인공신경망)

## 1. 개요

2017년 구글 브레인(현 Google DeepMind) 연구진이 논문 *Attention Is All You Need*에서 발표한 인공신경망 구조입니다. 기존의 순환 신경망(RNN)과 합성곱 신경망(CNN)의 한계를 극복하며 등장했습니다.

주로 자연어 처리(NLP) 분야에서 혁신을 일으켰으며, "어텐션 메커니즘만으로 장거리 의존성 문제와 병렬 처리의 한계를 해결"하도록 설계되었습니다.

현재는 BERT, GPT, T5 등 수많은 고성능 언어 모델의 기반 구조가 되었으며, 자연어 처리뿐만 아니라 컴퓨터 비전, 음성 인식, 강화학습 등 다양한 분야에서 활용됩니다.

## 2. 등장배경

트랜스포머 이전 시퀀스-투-시퀀스 모델들은 주로 RNN, LSTM, GRU와 같은 순환 신경망 구조에 기반했으며, 다음과 같은 한계가 있었습니다:

### 2.1 장거리 의존성 문제
시퀀스의 길이가 길어질수록 초반부의 중요 정보가 후반부까지 제대로 전달되지 못하고 소실되거나 변질되는 현상을 의미합니다.

### 2.2 병렬 처리의 어려움
RNN 모델들이 순차적으로 계산하기 때문에 각 시점(단어)의 계산이 이전 시점의 계산 완료를 기다려야 하므로, GPU 활용 효율성이 낮습니다.

### 2.3 고정된 크기의 문맥 벡터 한계
인코더-디코더 구조에서 고정된 크기를 가진 문맥 벡터 하나로 입력 시퀀스 전체의 정보를 압축하면 정보 병목 현상이 발생합니다.

이를 해결하기 위해 **어텐션 메커니즘(attention mechanism)**이 도입되었습니다.

## 3. 어텐션 메커니즘

### 3.1 스케일드 닷-프로덕트 어텐션(Scaled Dot-Product Attention)

기본적인 어텐션 메커니즘으로, **쿼리(Query)**, **키(Key)**, **밸류(Value)**라는 세 가지 벡터를 기반으로 작동합니다.

수식:
```
Attention(Q,K,V) = softmax(QK^T / √d_k) V
```

성분 설명:
- **Q (쿼리)**: 현재 계산하려는 대상을 나타내는 벡터
- **K (키)**: 시퀀스 내의 모든 요소의 특징을 나타내는 벡터. 쿼리와의 유사도 계산에 사용
- **V (밸류)**: 시퀀스 내의 모든 요소의 실제 정보 값을 담은 벡터
- **QK^T**: 쿼리와 각 키 벡터 간의 내적을 통해 유사도 계산
- **√d_k**: 스케일링 인자. 키 벡터 차원의 제곱근으로 나누어 내적 값이 너무 커지는 것을 방지
- **softmax**: 계산된 유사도를 확률 분포로 변환하여 각 밸류 벡터에 대한 가중치 획득

### 3.2 멀티-헤드 어텐션(Multi-Head Attention)

단일 어텐션 함수 대신 여러 개의 '헤드'로 나누어 병렬적으로 수행한 후 결과를 결합합니다.

과정:
1. 쿼리, 키, 밸류를 h개의 서로 다른 선형 변환을 통해 부분 공간으로 투영
2. 각 헤드마다 독립적으로 스케일드 닷-프로덕트 어텐션 수행: `head_i = Attention(Q_i, K_i, V_i)`
3. 각 헤드의 결과를 모두 이어 붙임(concatenate)
4. 선형 변환(가중치 행렬 W^O)을 적용하여 최종 출력 획득

수식:
```
MultiHead(Q,K,V) = Concat(head_1,...,head_h)W^O
where head_i = Attention(QW^Q_i, KW^K_i, VW^V_i)
```

멀티-헤드 어텐션은 입력 시퀀스의 서로 다른 위치에 있는 정보들끼리 다양한 측면을 동시에 학습할 수 있게 해줍니다.

## 4. 구조

트랜스포머의 구조는 위치 인코딩(positional encoding), 멀티헤드 어텐션(Multi-Head Attention), FFN(순방향 신경망)으로 이루어져 있습니다.

최초로 제안된 트랜스포머는 기본적으로 **인코더-디코더 구조**를 따릅니다.

### 4.1 인코더(Encoder)

동일한 구조의 N개(원문에서는 N=6)의 레이어를 쌓아서 구성됩니다.

각 레이어의 서브 레이어:

#### 4.1.1 멀티-헤드 셀프-어텐션 (Multi-Head Self-Attention)
입력 시퀀스 내의 모든 단어 쌍 간의 관계를 파악합니다. 쿼리, 키, 밸류는 모두 이전 레이어의 출력(또는 첫 레이어의 경우 입력 임베딩 + 포지셔널 인코딩)으로부터 나옵니다.

#### 4.1.2 포지션-와이즈 피드-포워드 신경망 (Position-wise Feed-Forward Network)
각 위치마다 독립적으로 적용되는 완전 연결 신경망입니다.

수식:
```
FFN(x) = max(0, xW_1 + b_1)W_2 + b_2
```

레이어 정규화와 잔차 연결: 각 서브 레이어의 출력에는 잔차 연결(Residual Connection)과 계층 정규화(Layer Normalization)가 순차적으로 적용됩니다:
```
LayerNorm(x + Sublayer(x))
```

### 4.2 디코더(Decoder)

동일한 구조의 N개(논문에서는 N=6)의 레이어로 구성됩니다. 각 디코더 레이어는 인코더 레이어의 두 서브 레이어 외에 추가적으로 세 번째 서브 레이어를 가집니다.

#### 4.2.1 마스크드 멀티-헤드 셀프-어텐션 (Masked Multi-Head Self-Attention)
디코더의 입력(타겟 시퀀스)에 대해 셀프-어텐션을 수행합니다. 마스킹이 적용되므로, 특정 위치의 단어를 예측할 때 그 뒤에 오는 단어들의 정보(미래 정보)를 참고하지 않습니다.

#### 4.2.2 멀티-헤드 인코더-디코더 어텐션 (Multi-Head Encoder-Decoder Attention)
디코더가 인코더의 출력 정보를 활용하는 부분입니다. 쿼리는 이전 디코더 서브 레이어의 출력에서 오고, 키와 밸류는 인코더 스택의 최종 출력에서 나옵니다.

#### 4.2.3 포지션-와이즈 피드-포워드 신경망
인코더와 동일한 구조와 역할을 수행합니다.

디코더 최종 출력: 디코더 스택의 최종 출력은 선형 변환을 거치고 소프트맥스 함수를 통과하여 다음 단어에 대한 확률 분포를 생성합니다.

### 4.3 포지셔널 인코딩(Positional Encoding)

트랜스포머는 순환 구조가 없기 때문에 모델 자체적으로는 단어의 순서나 위치 정보를 절대로 알 수 없습니다. 이를 해결하기 위해 입력 임베딩 벡터에 포지셔널 인코딩을 더해줍니다.

*Attention Is All You Need*에서는 사인(sine) 함수와 코사인(cosine) 함수를 이용한 고정된 방식을 사용합니다:

수식:
```
PE_(pos,2i) = sin(pos / 10000^(2i/d_model))
PE_(pos,2i+1) = cos(pos / 10000^(2i/d_model))
```

여기서 d_model은 모델의 임베딩 차원입니다. 이 방식은 모델이 상대적인 위치 정보도 쉽게 학습할 수 있게 하고, 학습 데이터보다 긴 시퀀스에 대해서도 외삽(extrapolation)이 가능합니다.

### 4.4 Add & Norm

#### 4.4.1 잔차 연결 (Residual Connection)
각 서브 레이어의 입력 x를 해당 서브 레이어의 출력 Sublayer(x)에 더해주는 연산: `x + Sublayer(x)`. 이는 네트워크가 깊어져도 정보가 잘 전달되게 하고 학습을 용이하게 합니다.

#### 4.4.2 계층 정규화 (Layer Normalization)
각 레이어의 입력에 대해 정규화를 수행합니다. 배치 정규화(Batch Normalization)와 달리 각 데이터 샘플 내의 모든 특성(feature)에 걸쳐 평균과 분산을 계산합니다. 이는 미니배치 크기에 영향을 받지 않아 트랜스포머와 같은 모델에 더 적합합니다.

## 5. 파생 모델

### 5.1 BERT (Bidirectional Encoder Representations from Transformers)
트랜스포머의 인코더 구조만 사용하여 양방향 문맥을 학습하는 마스크 언어 모델. 다양한 다운스트림 NLP 태스크에서 범용적으로 뛰어난 성능을 보입니다.

### 5.2 GPT (Generative Pre-trained Transformer)
트랜스포머의 디코더 구조만 사용하여 단방향(왼쪽에서 오른쪽)으로 텍스트를 생성하는 모델. ChatGPT의 기반이 된 GPT-3, GPT-4 등이 여기에 속합니다.

### 5.3 T5 (Text-to-Text Transfer Transformer)
모든 NLP 문제를 텍스트-투-텍스트 형식으로 변환하여 해결하려는 통합적 접근 방식을 취한 모델. 인코더-디코더 구조를 사용합니다.

### 5.4 ViT (Vision Transformer)
트랜스포머 구조를 컴퓨터 비전 분야에 적용한 모델. 이미지를 여러 패치(patch)로 나누어 시퀀스처럼 처리하여 이미지 분류에서 CNN과 경쟁력 있는 성능을 보입니다.

기타 모델: XLNet, RoBERTa, ALBERT, BART, ELECTRA, Transformer-XL, Reformer, Longformer 등 수많은 변형 및 응용 모델들이 존재합니다.

## 6. 파급력

트랜스포머의 등장은 대규모 언어 모델(LLM) 시대를 열었으며 인공지능 연구 및 응용의 패러다임을 바꾸는 계기가 되었습니다.

트랜스포머의 병렬 처리 능력, 모델 확장성, 장거리 의존성 포착 능력은 타의 추종을 불허하며 BERT의 성공 이후 기존에 시계열 데이터 처리에 쓰이던 RNN의 입지를 대폭 약화시켰습니다.

전통적으로 CNN의 영역이던 컴퓨터 비전 영역에도 진출하여, 현재는 자연어 처리뿐만 아니라:
- 이미지 분석(ViT)
- 이미지와 영상 생성(DiT)
- 음성 인식(Whisper)
등 다양한 분야에 적용되고 있습니다.

단백질 구조 예측: 알파폴드2(AlphaFold 2)는 트랜스포머와 그래프 신경망(GNN)을 결합한 하이브리드 모델로, 데미스 허사비스에게 노벨화학상을 안겨주었습니다.

확산 모델 변화: 2023년을 기점으로 기존 U-net 기반 확산 모델들이 트랜스포머 백본인 DiT(Diffusion Transformer)로 대체되고 있습니다(Stable Diffusion 3, Sora).

## 7. 한계

### 7.1 계산 복잡도 문제
트랜스포머는 입력 시퀀스의 길이가 길어질수록 계산량과 메모리 사용량이 급격히 증가합니다. 핵심 메커니즘인 셀프-어텐션이 O(n² · d)의 계산 복잡도를 가지기 때문입니다(여기서 d는 임베딩 차원).

해결을 위해 Longformer, Reformer, Linformer 등 희소(sparse) 어텐션이나 효율적인 어텐션 메커니즘 연구가 활발히 진행 중입니다.

### 7.2 해석의 어려움
어텐션 가중치를 통해 모델이 어떤 부분에 집중하는지 시각화는 가능하지만, 현재 인공지능 연구자들조차 트랜스포머 모델의 결정 과정을 완전히 설명하지 못합니다.

Anthropic을 포함한 업계와 유수의 대학, 연구기관에서 트랜스포머라는 블랙박스를 글래스박스로 만드는 기술 개발에 힘쓰고 있습니다.

### 7.3 사전 학습 필요성
트랜스포머는 강력한 성능을 발휘하기 위해 대규모 데이터셋에서의 사전 학습(pre-training)이 거의 필수적입니다. 빅테크 기업들이 만든 LLM 프론티어 모델들은 매개변수는 있는대로 늘리고 데이터는 거의 전 인터넷을 박박 긁어다 쓰는 수준입니다.

### 7.4 추론 모델의 등장
이러한 한계를 극복하기 위해 최근에는 추론(Reasoning) 능력을 강조한 추론 모델이 등장하고 있습니다. o1 및 o3는 단순히 매개변수 확장이 아닌 복잡한 추론 능력 향상 방향으로 발전했으며, 이러한 추론 중심 접근이 차세대 트랜스포머 모델 개발의 주요 방향으로 떠오르고 있습니다.
